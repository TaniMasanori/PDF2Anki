# セットアップ自動化とレビュー用インストールの改善 (Setup Automation)

## 概要

TAやレビュー担当者がプロジェクトを簡単に実行できるように、セットアップと起動プロセスを自動化しました。これにより、手動でのコマンド入力や複雑な環境構築の手順を省略し、ワンクリック（または1つのコマンド）でアプリケーションを起動できるようになりました。

## 実施内容

### 1. 自動セットアップスクリプトの作成

以下のプラットフォーム向けに、セットアップから起動までを一括で行うスクリプトを作成しました：

- **Windows用**: `setup_and_run.bat`
- **Mac/Linux用**: `setup_and_run.sh`

これらのスクリプトは以下の処理を自動的に行います：
1.  **Python環境のチェック**: Python 3.10以上がインストールされているか確認。
2.  **Gitサブモジュールの更新**: `marker-api`などのサブモジュールを初期化・更新。
3.  **仮想環境の作成**: `venv`を作成し、アクティベート。
4.  **依存関係のインストール**: `requirements.txt`および`marker-api`の依存関係をインストール。バージョン競合（`transformers`など）も自動修正。
5.  **環境設定**: `.env`ファイルがない場合、テンプレートを作成（ユーザーにAPIキーの入力を促す）。
6.  **サーバーの起動**: Marker APIサーバーとStreamlitアプリを別々のウィンドウで起動。

### 2. 依存関係の競合解消

ルートの`requirements.txt`と`marker-api`サブモジュール間での`marker-pdf`バージョンの競合を解消しました。
- `requirements.txt`の`marker-pdf`のバージョン指定を更新し、互換性を確保しました。

### 3. ドキュメントの更新

`README.md`を更新し、冒頭に「Quick Start for Reviewers」セクションを追加しました。
- 自動スクリプトの使用方法を記載。
- 手動インストールの手順は「Legacy」として残し、詳細なトラブルシューティングが必要な場合のために保持。
- APIキーに関する注意書きを追加（必要に応じてメールで問い合わせる旨）。

## 今後の手順（ユーザー側）

1.  変更内容をGitにコミットしてください。
2.  提出前に、実際の環境（Windows等）で`setup_and_run.bat`を実行し、動作を確認することを推奨します。
3.  `.env`ファイルに正しいOpenAI APIキーが設定されていることを確認してください。
